_____Scenario for overriding_____ :
java
class Animal {
    void eat() {
        System.out.println("Animal eats food");
    }
}

class Dog extends Animal {
    // No eat() method here
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();  // Animal reference, Dog object
        myDog.eat();               // Output: Animal eats food
    }
}
🔍 What’s Happening Here?
Dog inherits the eat() method from Animal because it doesn’t define its own.

When you call myDog.eat(), Java looks for the method in the actual object (Dog), doesn’t find it, so it uses the inherited one from Animal.

Since eat() is not overridden, the parent class version is used.

🧠 So What’s the Use?
Even though Dog doesn’t override eat(), inheritance still provides value:

Code Reuse: You don’t have to rewrite eat() in every subclass unless the behavior needs to change.

Consistency: All animals (Dog, Cat, etc.) can share common behavior from the Animal class.
 
Flexibility: If you later decide that Dog needs a different eat() behavior, you can override it — and the rest of the code using Animal references will automatically use the new behavior.

_________________________________________Stream API:classes__________________________________________

-->A Stream is a sequence of elements(elements  means items inside your collection)

-->Unlike Collections, Streams don’t store data, they process data.
-->Using collections also we can process data but entair structure will be changed but when we use  Streams mainly Process data without changing the original source.

-->It supports sequential and parallel aggregate operations.
    sequential:
        1)Run on one thread.
        2)Process elements one by one, in order.
        3)Default when you call .stream().

    Parallel Aggregate Operations:
        1)Run on multiple threads simultaneously.
        2)Created using .parallelStream() or .stream().parallel().
        3)Splits the data into chunks, processes them in parallel, then merges results.


Java Stream Pipeline:
Data Source  →  Intermediate Operations  →  Terminal Operation
(Collection,     (filter, map, sorted,      (forEach, collect,
 Array, I/O)      distinct, limit, etc.)     reduce, count, etc.)


Visual Example

 [ "John", "Jane", "Jack", "Tom" ]   // Data Source (List)

        ↓ stream()

        filter(name -> name.startsWith("J"))   // Intermediate
        ↓
        map(String::toUpperCase)               // Intermediate
        ↓
        sorted()                               // Intermediate
        ↓
        forEach(System.out::println)           // Terminal


Code Example with Pipeline

import java.util.Arrays;
import java.util.List;

public class StreamPipelineExample {
    public static void main(String[] args) {
        List<String> names = Arrays.asList("John", "Jane", "Jack", "Tom");

        names.stream() // Source
             .filter(name -> name.startsWith("J")) // Intermediate
             .map(String::toUpperCase)             // Intermediate
             .sorted()                             // Intermediate
             .forEach(System.out::println);        // Terminal
    }
}

Output:

JACK
JANE
JOHN

Pipeline Rules
Source → Can be a collection, array, or generated data.
Intermediate operations → Transform or filter data, return a new stream, lazy execution.
Terminal operation → Produces a result or side-effect, triggers execution.
Stream is consumed once → You can’t reuse it after a terminal operation.
